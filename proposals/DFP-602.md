## 摘要

本提案定义 Dovetail 的插件系统架构、注册机制、接口规范和生命周期管理。插件系统允许第三方开发者通过注册后端、优化器等组件来扩展编译器功能，而无需修改编译器核心代码。

---

## 动机

### 问题陈述

1. **编译器功能扩展困难**
    - 新增后端需要修改核心代码
    - 优化器无法独立开发和测试
    - 功能模块与编译器紧耦合

2. **缺少标准化扩展机制**
    - 第三方开发者无规范可循
    - 不同扩展之间无法互操作
    - 版本兼容性难以保证

3. **职责边界不清晰**
    - 编译器核心功能边界模糊
    - 无法灵活组合不同工具链组件

### 设计目标

- **模块化**：编译器核心最小化，通过插件注册扩展功能
- **统一接口**：所有扩展通过 `plugin_api` 统一接入
- **灵活性**：单个插件可注册多种类型的组件
- **可组合性**：插件之间可以互相依赖和协作

---

## 核心概念

### 1. 插件与组件

**插件（Plugin）**：

- 插件是扩展的容器，负责向编译器注册各种组件
- 单个插件可以注册多个后端、多个优化器等
- 插件本身不限定类型

**组件（Component）**：

- 组件是具体的功能实现，如后端、优化器、构建工具等
- 组件由插件在初始化时注册到编译器
- 编译器通过组件管理器调度各组件

**关系**：

```
插件 (Plugin)
  ├─ 注册后端 A (Backend)
  ├─ 注册后端 B (Backend)
  ├─ 注册优化器 X (Optimizer)
  └─ 注册优化器 Y (Optimizer)
```

### 2. Plugin API

**Plugin API** 是编译器提供的核心接口层：

- 所有插件通过 `plugin_api` 与编译器交互
- `plugin_api` 本身也以插件形式实现，保证扩展性
- 提供统一的注册、配置、事件等服务

## 技术规范

### 1. 插件发现与加载

#### 1.1 插件位置

插件**仅**存放在编译器安装目录下：

```
<dovetail_install_dir>/
└── plugins/
    ├── plugin_official_backends/     # 官方后端插件
    ├── plugin_core_optimizers/       # 核心优化器插件
    ├── plugin_api/                   # Plugin API 插件
    └── plugin_third_party/           # 第三方插件
```

**设计原则**：

- **全局唯一**：避免路径搜索的复杂性和冲突
- **集中管理**：便于版本控制和依赖解析
- **权限隔离**：插件与用户代码分离

#### 1.2 插件目录结构

**最小结构**：

```
<plugin_name>/
└── plugin.toml          # 必需：插件元数据
```

**关键点**：

- 不强制要求 `__init__.py`
- 实际入口由 `plugin.toml` 中 `[plugin].entry` 指定
- 单个插件可包含多个模块文件

### 2. 插件元数据

#### 2.1 `plugin.toml` 格式

```toml
[plugin]
name = "official_backends"                # 插件唯一标识（必需）
version = "1.0.0"                         # 插件版本（必需，SemVer）
api_version = "1.0"                       # Plugin API 版本（必需）
entry = "plugin_official_backends.loader" # 入口模块路径（必需）

[metadata]
author = "Dovetail Team"                # 作者（可选）
description = "Official backend collection"  # 描述（可选）
license = "MIT"                         # 许可证（可选）
homepage = "https://dovetail.dev"       # 主页（可选）

[dependencies]
# 依赖的其他插件（可选）
# plugin_core_optimizers = "^1.0.0"

[compatibility]
dovetail_min = "0.5.0"                  # 最低编译器版本（可选）
dovetail_max = "1.0.0"                  # 最高编译器版本（可选）

[config]
# 插件默认配置（可选）
# enable_experimental = false
```

**字段说明**：

| 字段                   | 类型     | 必需 | 说明                        |
|----------------------|--------|----|---------------------------|
| `plugin.name`        | String | 是  | 插件唯一标识符（小写字母、数字、下划线、入口模块） |
| `plugin.version`     | String | 是  | 插件版本（语义化版本）               |
| `plugin.api_version` | String | 是  | 依赖的 Plugin API 版本         |
| `metadata.*`         | String | 否  | 插件元信息                     |
| `dependencies.*`     | String | 否  | 依赖的其他插件及版本约束              |
| `compatibility.*`    | String | 否  | 编译器版本兼容性约束                |
| `config.*`           | Any    | 否  | 插件默认配置                    |

#### 2.2 入口函数规范

**签名**：

```python
def register(api: PluginAPI) -> bool:
    """
    插件注册入口函数
    
    Args:
        api: Plugin API 实例
    
    Returns:
        True 表示注册成功，False 表示失败
    """
    pass
```

**职责**：

1. 注册后端、优化器等组件到编译器
2. 注册编译器事件
3. 初始化插件内部状态
4. 验证运行环境

**示例**：

```python
def register(api):
    # 注册多个后端
    api.register_backend(JE1214Backend())
    api.register_backend(JE1214BetaBackend())
    api.register_backend(BedrockBackend())

    # 注册事件
    api.register_event('post_parse', on_parse_complete)

    # 初始化配置
    api.set_config('default_backend', 'je1214')

    return True
```

### 3. Plugin API 规范

#### 3.1 Plugin API 接口

Plugin API 提供以下核心服务：

**插件继承基类**：

```python

class Plugin(ABC):
    def __init__(self): ...

    @abstractmethod
    def load(self): ...

    @abstractmethod
    def unload(self) -> bool: ...

    @abstractmethod
    def validate(self) -> tuple[bool, str | None]: ...

    def initialize(self) -> None: ...

    def get_memory_usage(self) -> int: ...

    def handle_message(self, sender: Plugin, message: Any) -> Any: ...

    def send_message(self, target: str | Plugin, message: Any) -> bool: ...
```

**组件注册接口**：

```python
 # 注册后端
def registry_backend(cls, backend_class: type[Backend]) -> None: ...


# 注册优化管道
def registry_optimize_pass(self, pass_class: type[IROptimizationPass]) -> None: ...


# 注册内置库
def registry_library(cls, name: str, lib: type[Library]):


# 注册编译器注解
def registry_annotation(name: str, annotation: Annotation) -> bool:
```

**配置接口**：

```
 # 获取配置值
 fn get_config(key: String, default: Any = None) -> Any
 
 # 设置配置值
 fn set_config(key: String, value: Any) -> Void
 
 # 获取编译器信息
 property compiler_version: String
 property install_dir: Path
 property api_version: String
```

#### 3.2 Plugin API 加载

**Plugin API 本身是插件**：

```
plugins/
└── plugin_api/
    ├── plugin.toml
    └── api.py
```

**plugin_api/plugin.toml**：

```toml
[plugin]
name = "plugin_api"
version = "1.0.0"
api_version = "1.0"

[entry]
module = "plugin_api.api"
function = "bootstrap"
```

**优势**：

- Plugin API 本身可被替换或扩展
- 保证系统的一致性和可扩展性
- 允许用户自定义插件加载机制

### 4. 后端注册机制

#### 4.1 后端接口

后端是独立的类，由插件注册到编译器：

```python
class Backend(ABC, metaclass=BackendMeta):
    # 核心组件
    processor_registry: ProcessorRegistry = None
    output_manager: OutputManager = None

    def __init__(self, ir_builder: IRBuilder, target: Path, config: CompileConfig): ...

    @staticmethod
    @abstractmethod
    def is_support(config: CompileConfig) -> bool: ...

    @staticmethod
    @abstractmethod
    def get_name() -> str: ...

    def generate(self): ...

    def _process_instructions(self, context: GenerationContext): ...

    def _write_outputs(self, context: GenerationContext): ...
```

#### 4.2 后端注册

**注册流程**：

```python
# 插件中定义后端类
class JE1214Backend(Backend):
    def get_name(self):
        return "je1214"

    def is_support(self):
        return True


# 在入口函数中注册
def register(api):
    api.register_backend(JE1214Backend)
    return True
```

**编译器调用**：

```python
# 编译器通过后端管理器获取后端
backend = backend_manager.get_backend("je1214")  # 伪代码，实际管理器由编译器自行实现
result = backend.generate(ir, output_dir)
```

### 6. 事件系统

#### 6.1 标准事件

| 事件名称            | 触发时机   | 传递参数                               |
|-----------------|--------|------------------------------------|
| `pre_parse`     | 解析文件前  | `file_path: Path`                  |
| `post_parse`    | 解析文件后  | `ast: AST`                         |
| `pre_semantic`  | 语义分析前  | `ast: AST`                         |
| `post_semantic` | 语义分析后  | `ast: AST, symbols: SymbolTable`   |
| `pre_ir_gen`    | IR 生成前 | `ast: AST`                         |
| `post_ir_gen`   | IR 生成后 | `ir: IR`                           |
| `pre_optimize`  | 优化前    | `ir: IR, level: OptimizationLevel` |
| `post_optimize` | 优化后    | `ir: IR`                           |
| `pre_codegen`   | 代码生成前  | `ir: IR, backend: Backend`         |
| `post_codegen`  | 代码生成后  | `result: BackendResult`            |

#### 6.2 事件注册

```python
def on_parse_complete(ast):
    print(f"Parsed AST with {ast.node_count()} nodes")


def register(api):
    api.register_event('post_parse', on_parse_complete)
    api.register_event('pre_optimize', on_pre_optimize)
    return True
```

**调用顺序**：

- 同一事件的多个回调按注册顺序调用
- 回调不应修改传入的数据结构（除非明确允许）
- 回调抛出异常时，编译器应记录错误并继续

### 7. 插件生命周期

#### 7.1 加载顺序

1. 编译器硬编码加载 `plugin_loader` 插件
2. `plugin_loader` 插件初始化插件命名空间并加载 `plugin_api` 和其他插件
3. `plugin_api` 初始化核心服务（注册表、事件系统等）
4. `plugin_api` 提供接口供其他插件使用
5. 其他插件通过 `plugin_api` 注册组件


```
编译器启动
└─> 加载 plugin_loader 插件（硬编码）
   └─> 扫描插件目录
      └─> 读取所有 plugin.toml
      └─> 解析依赖关系
      └─> 拓扑排序
         └─> 加载 plugin_api 插件和其他插件
            └─> plugin_api 初始化核心服务
         └─> 按依赖顺序加载插件
            └─> 导入入口模块
            └─> 调用入口函数 register(api)
            └─> 插件注册组件和钩子
               └─> 编译器就绪，执行编译任务
```

#### 7.2 插件依赖

**依赖声明**：

```toml
[dependencies]
plugin_core_optimizers = "^1.0.0"
plugin_api = "^1.0.0"
```

**依赖解析**：

- 编译器按拓扑顺序加载插件
- 被依赖的插件先于依赖者加载
- 循环依赖应被检测并报错

**缺失依赖处理**：

```
error: plugin 'my_plugin' depends on 'plugin_core_optimizers', which is not installed
  --> my_plugin/plugin.toml:8:1
   |
 8 | plugin_core_optimizers = "^1.0.0"
   | ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ required dependency not found
   |
   = help: install the dependency with: dovetail plugin install plugin_core_optimizers
```

### 8. 版本兼容性

#### 8.1 Plugin API 版本

Plugin API 使用语义化版本：

```
主版本.次版本
```

**兼容性规则**：

- 插件 API 主版本必须与当前 Plugin API 主版本相同
- 插件 API 次版本不得高于当前 Plugin API 次版本

**示例**：

```
当前 Plugin API 版本：2.3

兼容插件：2.0, 2.1, 2.2, 2.3
不兼容插件：1.x, 2.4, 3.x
```

#### 8.2 编译器版本约束

插件可限制支持的编译器版本：

```toml
[compatibility]
dovetail_min = "0.5.0"
dovetail_max = "1.0.0"
```

**约束检查**：

```
if compiler_version < dovetail_min or compiler_version > dovetail_max:
    skip_plugin()
    log_warning("Plugin incompatible with current compiler version")
```

### 9. 配置系统

#### 9.1 配置来源

配置优先级（从高到低）：

1. **命令行参数**：`--config key=value`
2. **环境变量**：`DOVETAIL_CONFIG_KEY=value`
3. **项目配置文件**：`dovetail.toml`
4. **插件默认配置**：`plugin.toml [config]`
5. **编译器默认值**

#### 9.2 插件配置

**插件默认配置**：

```toml
# plugin.toml
[config]
backend_default = "je1214"
enable_experimental = false
optimization_threshold = 100
```

**用户覆盖**：

```toml
# dovetail.toml
[plugins.official_backends]
backend_default = "bedrock"
enable_experimental = true
```

**访问配置**：

```python
def register(api):
    backend = api.get_config('backend_default', 'je1214')
    experimental = api.get_config('enable_experimental', False)

    if experimental:
        api.register_backend(ExperimentalBackend())
```

### 11. 性能考虑

#### 11.1 组件调用性能

**性能预算**：

- 单优化管道开销 < 编译时间的 30%
- 后端生成时间与 IR 长度成线性关系
- 钩子回调应轻量，避免阻塞编译流程

#### 11.2 内存管理

**建议**：

- 插件应及时释放不再使用的资源
- 避免在全局变量中缓存大型数据
- 使用弱引用避免循环引用
