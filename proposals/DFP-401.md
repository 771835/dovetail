## 摘要

本提案规范 Dovetail 的文件包含机制（`include` 语句）的语法、语义和路径解析规则，以及推荐的项目目录结构约定。构建工具、依赖管理等编译工具链职责由独立的构建系统插件承担，不在本提案范围内。

---

## 动机

### 问题陈述

1. **`include` 语义不明**
    - 路径解析规则不明确
    - 重复包含行为未规范
    - 循环依赖检测缺失

2. **项目组织无约定**
    - 标准库、用户代码、测试代码无明确边界
    - 大型项目缺少组织指导

3. **编译器职责模糊**
    - 编译器应负责哪些路径搜索？
    - 哪些职责应交由外部工具？

### 设计目标

- **最小核心**：编译器仅负责 `include` 语句的基本语义
- **清晰边界**：区分编译器职责与构建工具职责
- **简洁语法**：沿用 C 风格的包含机制，符合 Dovetail 的 Python/C 混合风格
- **可扩展性**：为构建工具插件预留扩展点

## 技术规范

### 1. `include` 语句

#### 1.1 语法定义

```lark
include_statement: "include" STRING
```

**示例：**

```mcdl
include "math"
include "utils/helpers"
include "./local_module"
include "../shared/config"
```

**约束：**

- `include` 为保留关键字
- 字符串字面量必须为编译时常量（不可为变量或表达式）
- 路径分隔符统一使用正斜杠 `/`（编译器负责平台转换）
- 路径字符串**必须包含** `.mcdl` 扩展名

#### 1.2 语义

**`include` 的作用：**

1. 将指定文件的 AST 合并到当前文件的解析过程中（类似 C 的 `#include`）
2. 在合并前，编译器处理重复包含和循环依赖
3. 被包含文件中的所有符号（函数、常量等）对包含者可见

**执行时机：**

- 编译器在解析阶段处理 `include` 语句
- 被包含文件先于当前文件完成语义分析
- 符号表在包含完成后对当前作用域可见

**等价性示例：**

```mcdl
// main.mcdl
include "utils"
fn main() {
    helper()
}

// utils.mcdl
fn helper() { ... }

// 语义等价于（概念上）：
fn helper() { ... }  // 来自 utils.mcdl
fn main() {
    helper()
}
```

#### 1.3 重复包含处理

**编译器行为规范：**

编译器**必须**维护已包含文件的规范路径集合，自动去重：

```mcdl
// a.mcdl
include "common"
include "b.mcdl"

// b.mcdl
include "common"  // ✅ 合法：编译器自动忽略（已被 a.mcdl 包含）
```

**实现要求：**

- 使用文件系统的**规范绝对路径**作为去重依据
- 解析符号链接的真实路径
- 跨平台路径比较时忽略大小写（Windows 兼容性）

**去重策略：**

```python
# 伪代码
included_files = set()


def process_include(file_path):
    canonical = file_path.resolve()  # 获取规范绝对路径
    if canonical in included_files:
        return  # 已包含，跳过
    included_files.add(canonical)
    # 继续处理文件...
```

#### 1.4 循环依赖检测

编译器**必须**检测并拒绝编译循环依赖：

```mcdl
// a.mcdl
include "b.mcdl"  // a → b

// b.mcdl
include "c.mcdl"  // b → c

// c.mcdl
include "a.mcdl"  // c → a  ❌ 循环：a → b → c → a
```

**实现要求：**

- 使用栈追踪当前包含链
- 检测到已在栈中的文件时报错
- 提供完整依赖路径供调试
- 栈帧应包含文件名(可选是否包含行号等信息)

### 2. 路径解析规则

#### 2.1 路径类型

Dovetail 的 `include` 支持三种路径类型：

| 类型       | 语法              | 示例            | 解析基准     |
|----------|-----------------|---------------|----------|
| **相对路径** | `./` 或 `../` 开头 | `"./utils"`   | 当前文件所在目录 |
| **绝对路径** | `/` 开头          | `"/lib/math"` | 项目根目录    |
| **搜索路径** | 无前缀             | `"math"`      | 搜索路径列表   |

#### 2.2 解析算法

编译器按以下顺序解析 `include` 路径：

**步骤 1：判断路径类型**

```python
def resolve_include(current_file: Path, include_path: str) -> Path:
    if include_path.startswith('./') or include_path.startswith('../'):
        # 相对路径
        return resolve_relative(current_file, include_path)
    elif include_path.startswith('/'):
        # 绝对路径（相对项目根）
        return resolve_absolute(project_root, include_path)
    else:
        # 搜索路径
        return resolve_search(include_path, search_paths)
```

**步骤 2：相对路径解析**

```python
def resolve_relative(current_file: Path, include_path: str) -> Path:
    """
    相对路径解析：相对于当前文件所在目录
    
    解析顺序：
    1. 若路径包含 .mcdl 扩展名，直接查找该文件
    2. 否则尝试追加 .mcdl 扩展名查找(非强制)
    3. 若路径指向目录，查找索引文件
    """
    base_dir = current_file.parent
    target = base_dir / include_path

    # 情况 1：显式指定扩展名
    if target.suffix == '.mcdl':
        if target.exists() and target.is_file():
            return target.resolve()
        raise FileNotFoundError(f"Cannot find {include_path}")

    # 情况 2：追加 .mcdl 扩展名
    target_with_ext = target.with_suffix('.mcdl')
    if target_with_ext.exists() and target_with_ext.is_file():
        return target_with_ext.resolve()

    # 情况 3：目录包含（可选特性）
    if target.exists() and target.is_dir():
        for index_name in ['__init__.mcdl', 'index.mcdl']:
            index_file = target / index_name
            if index_file.exists() and index_file.is_file():
                return index_file.resolve()

    # 所有尝试均失败
    raise FileNotFoundError(
        f"Cannot find {include_path} "
        f"(searched: {target_with_ext}, {target}/__init__.mcdl, {target}/index.mcdl)"
    )
```

**步骤 3：绝对路径解析**

```python
def resolve_absolute(project_root: Path, include_path: str) -> Path:
    """
    绝对路径解析：相对于项目根目录
    
    路径以 / 开头时去除前导斜杠后拼接到项目根
    """
    target = project_root / include_path.lstrip('/')

    # 同相对路径的解析逻辑
    if target.suffix == '.mcdl':
        if target.exists() and target.is_file():
            return target.resolve()
        raise FileNotFoundError(f"Cannot find {include_path}")

    target_with_ext = target.with_suffix('.mcdl')
    if target_with_ext.exists() and target_with_ext.is_file():
        return target_with_ext.resolve()

    if target.exists() and target.is_dir():
        for index_name in ['__init__.mcdl', 'index.mcdl']:
            index_file = target / index_name
            if index_file.exists() and index_file.is_file():
                return index_file.resolve()

    raise FileNotFoundError(f"Cannot find {include_path}")
```

**步骤 4：搜索路径解析**

```python
def resolve_search(include_path: str, search_paths: list[Path]) -> Path:
    """
    搜索路径解析：在搜索路径列表中依次查找
    
    搜索策略：
    1. 检查是否为内置库名称（无扩展名）
    2. 在每个搜索目录中尝试查找
    3. 支持带扩展名的直接文件名
    """
    attempted_paths = []

    # 情况 1：内置库名称（无扩展名）
    if not include_path.endswith('.mcdl'):
        for search_dir in search_paths:
            # 尝试查找同名模块文件
            module_file = search_dir / f"{include_path}.mcdl"
            attempted_paths.append(module_file)
            if module_file.exists() and module_file.is_file():
                return module_file.resolve()

            # 尝试查找同名目录的索引文件（可选特性）
            module_dir = search_dir / include_path
            if module_dir.exists() and module_dir.is_dir():
                for index_name in ['__init__.mcdl', 'index.mcdl']:
                    index_file = module_dir / index_name
                    attempted_paths.append(index_file)
                    if index_file.exists() and index_file.is_file():
                        return index_file.resolve()

    # 情况 2：包含扩展名的路径，直接查找文件
    for search_dir in search_paths:
        candidate = search_dir / include_path
        attempted_paths.append(candidate)
        if candidate.exists() and candidate.is_file():
            return candidate.resolve()

    # 所有搜索路径均未找到
    raise FileNotFoundError(
        f"Cannot find {include_path}\n"
        f"Searched in: {', '.join(str(p) for p in attempted_paths)}"
    )
```

#### 2.3 搜索路径列表

**默认搜索路径（编译器内置）：**

编译器**应**按以下优先级顺序搜索：

1. **标准库路径**：
    - 编译器安装目录下的 `lib/` 目录
    - 或通过命令行参数 `--lib-path <dir>` 显式指定
    - 或环境变量 `DOVETAIL_LIB_PATH` 指定

2. **当前文件目录**：
    - 正在处理的 `.mcdl` 文件所在目录

3. **项目源码路径**：
    - 项目根目录下的 `src/` 目录（若存在）
    - 由构建工具通过 `--project-root` 参数传递

4. **额外搜索路径**：
    - 通过命令行参数 `-I <dir>` 指定（可多次使用）
    - 由构建工具从配置文件读取并传递

**命令行参数示例：**

```bash
# 添加多个搜索路径
dovetail main.mcdl \
    --lib-path /usr/local/lib/dovetail \
    --project-root /home/user/my_project \
    -I /path/to/libs \
    -I ../shared
```

**搜索顺序完整示例：**

```
include "math"

搜索路径（按优先级）：
1. /usr/local/lib/dovetail/math.mcdl           # 标准库
2. /home/user/my_project/src/math.mcdl         # 当前文件目录
3. /home/user/my_project/src/math.mcdl         # 项目源码目录
4. /path/to/libs/math.mcdl                     # -I 指定的额外路径
5. /home/user/shared/math.mcdl                 # -I 指定的额外路径
```

**环境变量支持（可选）：**

```bash
# 设置标准库路径
export DOVETAIL_LIB_PATH=/opt/dovetail/lib

# 设置额外搜索路径（冒号分隔）
export DOVETAIL_INCLUDE_PATH=/path/to/libs:/path/to/shared

# 编译时自动读取
dovetail main.mcdl
```

#### 2.4 文件扩展名处理

**规范：**

- 用户在 `include` 语句中**必须**编写扩展名
- 编译器自动追加 `.mcdl` 扩展名是非标准做法

**示例：**

```mcdl
include "math.mcdl"         // ✅ 显式扩展名，直接查找 math.mcdl
include "utils/io.mcdl"     // ✅ 显式扩展名
```

**目录包含（可选特性）：**

若路径指向目录，编译器**可选**支持以下行为：

```mcdl
include "collections"

// 搜索顺序：
// 1. collections.mcdl（文件）
// 2. collections/__init__.mcdl（目录索引）
// 3. collections/index.mcdl（目录索引）
```

**优先级规则：**

```
文件 > 目录索引

即：
- 若 collections.mcdl 和 collections.mcdl/ 同时存在，优先包含 collections.mcdl
- 若仅存在 collections/ 目录，查找其索引文件
```

**内置库特例（可选）：**

对于标准库中的核心模块，编译器**可选**实现以下优化：

1. **内置符号注入**：
    - 部分核心函数（如 `abs`, `min`, `max`）无需通过文件包含
    - 直接在编译器中注册为内置符号

2. **虚拟模块映射**：
    - `include "math"` 可能不对应实际的 `math.mcdl` 文件
    - 而是编译器内部实现的虚拟模块

**实现建议（虚拟模块）：**

```python
# 内置虚拟模块映射
BUILTIN_VIRTUAL_MODULES = {
    'math': ['abs', 'min', 'max', 'clamp'],  # 内置函数列表
    'random': ['randint'],
}


def resolve_search(include_path: str, search_paths: list[Path]) -> Path:
    # 检查是否为虚拟模块
    if include_path in BUILTIN_VIRTUAL_MODULES:
        # 不返回文件路径，而是触发符号注入
        inject_builtin_symbols(include_path, BUILTIN_VIRTUAL_MODULES[include_path])
        return None  # 特殊标记：虚拟模块

    # 常规文件查找
    # ...
```

### 3. 项目结构约定

#### 3.1 推荐目录结构

本提案**推荐**（但不强制）以下项目结构：

```
my_project/
├── src/                    # 用户源代码
│   ├── main.mcdl           # 主入口文件
│   ├── game/
│   │   ├── player.mcdl
│   │   └── world.mcdl
│   └── utils/
│       └── helpers.mcdl
├── lib/                    # 第三方库（可选）
│   └── external_lib/
│       ├── __init__.mcdl
│       └── module.mcdl
├── tests/                  # 测试文件（可选）
│   ├── test_game.mcdl
│   └── test_utils.mcdl
├── docs/                   # 项目文档（可选）
│   └── README.md
├── hook/                   # 编译钩子（可选）
│   ├── pre_build.sh
│   └── post_build.sh
├── build/                  # 编译输出（工具生成）
│   └── datapack/
├── dovetail.toml           # 项目配置文件（可选）
└── README.md
```

#### 3.2 目录职责说明

| 目录       | 职责      | 约定                   |
|----------|---------|----------------------|
| `src/`   | 项目源代码   | 默认搜索路径之一             |
| `lib/`   | 第三方库/依赖 | 可通过 `-I lib` 添加到搜索路径 |
| `tests/` | 测试代码    | 由构建工具管理和执行，不参与生产编译   |
| `build/` | 编译产物    | 由构建工具管理，用户可忽略        |
| `docs/`  | 项目文档    | 不参与编译                |
| `hook/`  | 编译钩子    | 由构建工具在特定时机执行         |

#### 3.3 特殊文件约定

**主入口文件：**

- 默认为 `src/main.mcdl`
- 可通过命令行参数 `<input>` 指定
- 或由构建工具从配置文件读取

**模块索引文件（可选）：**

- 目录下的 `__init__.mcdl` 作为默认导出
- 支持 `include "collections"` 而非 `include "collections/stack.mcdl"`

**示例：**

```mcdl
// lib/collections/__init__.mcdl
include "collections/stack.mcdl"
include "collections/queue.mcdl"

// 用户代码
include "collections"  // 等价于包含 collections/__init__.mcdl
```

**编译钩子（可选）：**

- `hook/pre_build.sh` 或 `hook/pre_build.py`：编译前执行
- `hook/post_build.sh` 或 `hook/post_build.py`：编译后执行
- 由构建工具负责调用，编译器不处理
- 钩子脚本应具有可执行权限（Unix 系统）

#### 3.4 钩子系统（构建工具职责）

虽然钩子不由编译器处理，但本提案推荐以下约定：

**钩子触发时机：**

```
1. pre_build       - 整个构建开始前
2. pre_compile     - 单个文件编译前（每个文件触发一次）
3. post_compile    - 单个文件编译后
4. pre_optimize    - 优化阶段开始前
5. post_optimize   - 优化阶段结束后
6. pre_package     - 数据包打包开始前
7. post_build      - 整个构建结束后
```

**钩子文件命名规范：**

```
hook/
├── pre_build.sh       # Shell 脚本
├── pre_build.py       # Python 脚本
├── post_build.sh
└── custom_hook.sh     # 自定义钩子（需在配置文件中声明）
```

**钩子执行环境：**

构建工具应向钩子脚本传递以下环境变量：

```bash
DOVETAIL_PROJECT_ROOT   # 项目根目录
DOVETAIL_BUILD_DIR      # 构建输出目录
DOVETAIL_ENTRY_FILE     # 入口文件路径
DOVETAIL_BACKEND        # 后端名称
DOVETAIL_OPTIMIZATION   # 优化级别
```

**钩子文件格式示例：**

```bash
# hook/pre_build.sh
#!/bin/bash
set -e

echo "Starting build for project: $DOVETAIL_PROJECT_ROOT"
echo "Generating version file..."

# 生成版本文件
VERSION=$(git describe --tags --always)
echo "let VERSION = \"$VERSION\"" > src/version.mcdl

# 清理旧构建
rm -rf "$DOVETAIL_BUILD_DIR"/*
```

```python
# hook/post_build.py
# !/usr/bin/env python3
import os
import shutil
from pathlib import Path

project_root = Path(os.environ['DOVETAIL_PROJECT_ROOT'])
build_dir = Path(os.environ['DOVETAIL_BUILD_DIR'])

print(f"Build completed: {build_dir}")

# 复制到 Minecraft 世界目录
minecraft_world = Path.home() / '.minecraft/saves/MyWorld/datapacks'
if minecraft_world.exists():
    shutil.copytree(build_dir, minecraft_world / 'my_datapack', dirs_exist_ok=True)
    print(f"Datapack installed to: {minecraft_world}")
```

### 4. 编译器职责边界

#### 4.1 编译器必须实现的功能

1. **`include` 语法解析**
    - 识别 `include` 语句
    - 提取路径字符串
    - 验证路径格式合法性

2. **路径解析**
    - 按本提案 2.2 规定的算法解析路径
    - 处理相对/绝对/搜索路径
    - 规范化路径为绝对路径

3. **重复包含去重**
    - 维护已包含文件集合
    - 自动忽略重复包含
    - 跨平台路径比较

4. **循环依赖检测**
    - 维护包含栈
    - 检测并报告循环依赖
    - 提供完整依赖链

5. **错误报告**
    - 文件未找到错误（含搜索路径列表）
    - 循环依赖错误（含依赖链）
    - 路径格式错误
    - 提供清晰的错误位置（文件名、行号、列号）

6. **命令行参数处理**
    - `-I <dir>`：添加搜索路径
    - `--lib-path <dir>`：指定标准库路径
    - `--project-root <dir>`：指定项目根目录
    - `<input>`：指定入口文件

#### 4.2 编译器不负责的功能

以下职责应由**外部构建工具**承担：

1. **项目配置管理**
    - 读取 `dovetail.toml` 配置文件
    - 解析构建参数
    - 管理项目元数据

2. **依赖管理**
    - 下载远程依赖（Git、HTTP）
    - 版本解析和冲突检测
    - 依赖锁定文件（lock file）

3. **增量编译**
    - 检测文件修改时间戳
    - 维护编译缓存
    - 仅重编译变更文件及其依赖

4. **并行编译**
    - 多文件并行处理

## 附录 A：编译器实现参考

### include_manager.py 完整实现

```python
from pathlib import Path
from typing import Set, List, Optional

# 内置虚拟模块白名单（可选特性）
BUILTIN_VIRTUAL_MODULES = {
    'math': ['abs', 'min', 'max', 'clamp'],
    'random': ['randint'],
}


class IncludeManager:
    """管理文件包含的核心组件"""

    def __init__(
            self,
            project_root: Path,
            stdlib_path: Path,
            search_paths: List[Path]
    ):
        """
        初始化包含管理器
        
        Args:
            project_root: 项目根目录
            stdlib_path: 标准库路径
            search_paths: 额外搜索路径列表
        """
        self.project_root = project_root.resolve()
        self.stdlib_path = stdlib_path.resolve()

        # 构建完整搜索路径列表（顺序很重要）
        self.search_paths = [
            self.stdlib_path,  # 1. 标准库
            # 当前文件目录在 resolve() 中动态添加
            # 2. 项目源码目录（若存在）
            *(self.project_root / 'src',) if (self.project_root / 'src').exists() else (),
            # 3. 额外搜索路径
            *[Path(p).resolve() for p in search_paths]
        ]

        self._included_files: Set[Path] = set()
        self._include_stack: List[Path] = []

    def resolve(self, current_file: Path, include_path: str) -> Optional[Path]:
        """
        解析 include 路径为绝对路径
        
        Args:
            current_file: 当前正在处理的文件路径
            include_path: include 语句中的路径字符串
        
        Returns:
            解析后的规范绝对路径，若为虚拟模块则返回 None
        
        Raises:
            FileNotFoundError: 文件未找到
        """
        # 规范化路径分隔符
        include_path = include_path.replace('\\', '/')

        # 相对路径
        if include_path.startswith('./') or include_path.startswith('../'):
            return self._resolve_relative(current_file, include_path)

        # 绝对路径
        if include_path.startswith('/'):
            return self._resolve_absolute(include_path)

        # 搜索路径
        return self._resolve_search(current_file, include_path)

    def should_include(self, file_path: Optional[Path]) -> bool:
        """
        判断文件是否应该包含（去重检查）
        
        Args:
            file_path: 文件的规范绝对路径（若为虚拟模块则为 None）
        
        Returns:
            True 若文件未被包含过，False 否则
        """
        if file_path is None:  # 虚拟模块
            return True

        canonical = file_path.resolve()
        return canonical not in self._included_files

    def mark_included(self, file_path: Optional[Path]):
        """标记文件已包含"""
        if file_path is not None:
            self._included_files.add(file_path.resolve())

    def enter_file(self, file_path: Optional[Path]):
        """
        进入文件处理（用于循环依赖检测）
        
        Args:
            file_path: 文件路径（虚拟模块为 None）
        
        Raises:
            CircularDependencyError: 检测到循环依赖
        """
        if file_path is None:  # 虚拟模块不参与循环检测
            return

        canonical = file_path.resolve()

        if canonical in self._include_stack:
            chain = self._include_stack + [canonical]
            raise CircularDependencyError(chain)

        self._include_stack.append(canonical)

    def exit_file(self, file_path: Optional[Path]):
        """退出文件处理"""
        if file_path is not None and self._include_stack:
            self._include_stack.pop()

    # 私有方法
    def _resolve_relative(self, current_file: Path, path: str) -> Path:
        """相对路径解析"""
        base_dir = current_file.parent.resolve()
        target = base_dir / path

        # 显式扩展名
        if target.suffix == '.mcdl':
            if target.exists() and target.is_file():
                return target.resolve()
            raise FileNotFoundError(f"Cannot find {path}")

        # 自动追加扩展名
        target_with_ext = target.with_suffix('.mcdl')
        if target_with_ext.exists() and target_with_ext.is_file():
            return target_with_ext.resolve()

        # 目录包含
        if target.exists() and target.is_dir():
            for index_name in ['__init__.mcdl', 'index.mcdl']:
                index_file = target / index_name
                if index_file.exists() and index_file.is_file():
                    return index_file.resolve()

        raise FileNotFoundError(
            f"Cannot find {path}\n"
            f"Searched: {target_with_ext}, {target}/__init__.mcdl, {target}/index.mcdl"
        )

    def _resolve_absolute(self, path: str) -> Path:
        """绝对路径解析"""
        target = self.project_root / path.lstrip('/')

        if target.suffix == '.mcdl':
            if target.exists() and target.is_file():
                return target.resolve()
            raise FileNotFoundError(f"Cannot find {path}")

        target_with_ext = target.with_suffix('.mcdl')
        if target_with_ext.exists() and target_with_ext.is_file():
            return target_with_ext.resolve()

        if target.exists() and target.is_dir():
            for index_name in ['__init__.mcdl', 'index.mcdl']:
                index_file = target / index_name
                if index_file.exists() and index_file.is_file():
                    return index_file.resolve()

        raise FileNotFoundError(f"Cannot find {path}")

    def _resolve_search(self, current_file: Path, path: str) -> Optional[Path]:
        """搜索路径解析"""
        # 虚拟模块检测
        if path in BUILTIN_VIRTUAL_MODULES:
            # 返回 None 表示这是虚拟模块，需要符号注入
            return None

        # 动态添加当前文件目录到搜索路径
        current_dir = current_file.parent.resolve()
        search_paths_with_current = [current_dir] + self.search_paths

        attempted_paths = []

        # 无扩展名：尝试多种可能
        if not path.endswith('.mcdl'):
            for search_dir in search_paths_with_current:
                # 同名文件
                module_file = search_dir / f"{path}.mcdl"
                attempted_paths.append(module_file)
                if module_file.exists() and module_file.is_file():
                    return module_file.resolve()

                # 同名目录的索引
                module_dir = search_dir / path
                if module_dir.exists() and module_dir.is_dir():
                    for index_name in ['__init__.mcdl', 'index.mcdl']:
                        index_file = module_dir / index_name
                        attempted_paths.append(index_file)
                        if index_file.exists() and index_file.is_file():
                            return index_file.resolve()

        # 带扩展名：直接查找
        for search_dir in search_paths_with_current:
            candidate = search_dir / path
            attempted_paths.append(candidate)
            if candidate.exists() and candidate.is_file():
                return candidate.resolve()

        # 所有尝试均失败
        raise FileNotFoundError(
            f"Cannot find {path}\n"
            f"Searched in:\n" + "\n".join(f"  - {p}" for p in attempted_paths)
        )


class CircularDependencyError(Exception):
    """循环依赖错误"""

    def __init__(self, chain: List[Path]):
        self.chain = chain
        super().__init__(self._format_chain())

    def _format_chain(self) -> str:
        names = [p.name for p in self.chain]
        return " -> ".join(names)
```
