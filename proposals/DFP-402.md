## 摘要

本提案规范 Dovetail 的文件包含机制（`include` 语句）的语法、语义和路径解析规则，以及推荐的项目目录结构约定。构建工具、依赖管理等编译工具链职责由独立的构建系统插件承担，不在本提案范围内。

---

## 动机

### 问题陈述

1. **`include` 语义不明**
    - 路径解析规则不明确
    - 重复包含行为未规范
    - 循环依赖检测缺失

2. **项目组织无约定**
    - 标准库、用户代码、测试代码无明确边界
    - 大型项目缺少组织指导

3. **编译器职责模糊**
    - 编译器应负责哪些路径搜索？
    - 哪些职责应交由外部工具？

### 设计目标

- **最小核心**：编译器仅负责 `include` 语句的基本语义
- **清晰边界**：区分编译器职责与构建工具职责
- **简洁语法**：沿用 C 风格的包含机制，符合 Dovetail 的 Python/C 混合风格
- **可扩展性**：为构建工具插件预留扩展点

## 技术规范

### 1. `include` 语句

#### 1.1 语法定义

```lark
include: "include" literal
```

**示例：**

```mcdl
include "math"
include "utils/helpers"
include "./local_module"
include "../shared/config"
```

**约束：**

- `include` 为保留关键字
- 字符串字面量必须为编译时常量（不可为变量或表达式）
- 路径分隔符统一使用正斜杠 `/`（编译器负责平台转换）
- 路径字符串**不包含** `.mcdl` 扩展名

#### 1.2 语义

**`include` 的作用：**

1. 将指定文件的内容**文本级别插入**当前位置（类 C 的 `#include`）
2. 在插入前，编译器处理重复包含和循环依赖
3. 被包含文件中的所有符号（函数、常量等）对包含者可见

**等价性示例：**

```mcdl
// main.mcdl
include "utils"
fn main() {
    helper()
}

// utils.mcdl
fn helper() { ... }

// 等价于（概念上）：
fn helper() { ... }  // 来自 utils.mcdl
fn main() {
    helper()
}
```

#### 1.3 重复包含处理

**编译器行为规范：**

编译器**必须**维护已包含文件的规范路径集合，自动去重：

```mcdl
// a.mcdl
include "common"
include "b"

// b.mcdl
include "common"  // ✅ 合法：编译器自动忽略（已被 a.mcdl 包含）
```

**实现要求：**

- 使用文件系统的**规范绝对路径**作为去重依据
- 解析符号链接的真实路径

#### 1.4 循环依赖检测

编译器**必须**检测并拒绝编译循环依赖：

```mcdl
// a.mcdl
include "b"  // a → b

// b.mcdl
include "c"  // b → c

// c.mcdl
include "a"  // c → a  ❌ 循环：a → b → c → a
```

**错误报告规范：**

```
error: circular dependency detected
  --> c.mcdl:1:9
   |
 1 | include "a"
   |         ^^^ creates circular dependency
   |
   = note: dependency chain:
     a.mcdl -> b.mcdl -> c.mcdl -> a.mcdl
```

**实现要求：**

- 使用栈追踪当前包含链
- 检测到已在栈中的文件时报错
- 提供完整依赖路径供调试

### 2. 路径解析规则

#### 2.1 路径类型

Dovetail 的 `include` 支持三种路径类型：

| 类型       | 语法              | 示例                 | 解析基准     |
|----------|-----------------|--------------------|----------|
| **相对路径** | `./` 或 `../` 开头 | `"./utils.mcdl"`   | 当前文件所在目录 |
| **绝对路径** | `/` 开头          | `"/lib/math.mcdl"` | 项目根目录    |
| **搜索路径** | 无前缀             | `"math.mcdl"`      | 搜索路径列表   |

#### 2.2 解析算法

编译器按以下顺序解析 `include` 路径：

**步骤 1：判断路径类型**

```python
if path.startswith('./') or path.startswith('../'):
    # 相对路径
    return resolve_relative(current_file, path)
elif path.startswith('/'):
    # 绝对路径（相对项目根）
    return resolve_absolute(project_root, path)
else:
    # 搜索路径
    return resolve_search(path, search_paths)
```

**步骤 2：相对路径解析**

```python
def resolve_relative(current_file: Path, include_path: str) -> Path:
    base_dir = current_file.parent
    target = base_dir / include_path

    # 若路径包含扩展名，直接使用
    if target.suffix == '.mcdl':
        if not target.exists():
            raise FileNotFoundError(f"Cannot find {include_path}")
        return target.resolve()

    # 若为目录，尝试查找索引文件（可选特性）
    if target.is_dir():
        for index_name in ['__init__.mcdl', 'index.mcdl']:
            index_file = target / index_name
            if index_file.exists():
                return index_file.resolve()

    raise FileNotFoundError(f"Cannot find {include_path}")
```

**步骤 3：绝对路径解析**

```python
def resolve_absolute(project_root: Path, include_path: str) -> Path:
    target = project_root / include_path.lstrip('/')

    if target.suffix == '.mcdl':
        if not target.exists():
            raise FileNotFoundError(f"Cannot find {include_path}")
        return target.resolve()

    if target.is_dir():
        for index_name in ['__init__.mcdl', 'index.mcdl']:
            index_file = target / index_name
            if index_file.exists():
                return index_file.resolve()

    raise FileNotFoundError(f"Cannot find {include_path}")
```

**步骤 4：搜索路径解析**

```python
def resolve_search(include_path: str, search_paths: list[Path]) -> Path:
    # 首先检查是否为内置库名称（无扩展名）
    if not include_path.endswith('.mcdl'):
        for search_dir in search_paths:
            # 尝试查找同名模块文件
            module_file = search_dir / f"{include_path}.mcdl"
            if module_file.exists():
                return module_file.resolve()

            # 尝试查找同名目录的索引文件（可选特性）
            module_dir = search_dir / include_path
            if module_dir.is_dir():
                for index_name in ['__init__.mcdl', 'index.mcdl']:
                    index_file = module_dir / index_name
                    if index_file.exists():
                        return index_file.resolve()

    # 包含扩展名的情况，直接查找文件
    for search_dir in search_paths:
        candidate = search_dir / include_path
        if candidate.exists():
            return candidate.resolve()

    raise FileNotFoundError(
        f"Cannot find {include_path} in search paths: {search_paths}"
    )
```

#### 2.3 搜索路径列表

**默认搜索路径（编译器内置）：**

编译器**应**按以下顺序搜索：

1. **标准库路径**：编译器安装目录下的 `lib/` 目录
2. **当前项目路径**：项目根目录下的 `src/` 目录（若存在）
3. **额外搜索路径**：通过命令行参数 `-I` 或构建工具指定

**示例：**

```bash
# 添加额外搜索路径
dovetail compile main.mcdl -I /path/to/libs -I ../shared
```

**搜索顺序示例：**

```
include "math.mcdl"

搜索路径（按优先级）：
1. <dovetail_install_dir>/lib/math.mcdl
2. <project_root>/src/math.mcdl
3. /path/to/libs/math.mcdl
4. ../shared/math.mcdl
```

#### 2.4 文件扩展名处理

**规范：**

- 用户在 `include` 语句中**必须**编写完整扩展名
- 编译器不自动追加扩展名

**示例：**

```mcdl
include "math.mcdl"         // ✅ 正确
include "utils/io.mcdl"     // ✅ 正确
include "logger"            // ❌ 错误：缺少扩展名
```

**目录包含（可选特性）：**

若路径指向目录，编译器**可选**支持以下行为：

```mcdl
include "collections"

// 搜索顺序：
// 1. 内置库 collections（若存在名为 collections.mcdl 的内置模块）
// 2. collections/__init__.mcdl（若存在）
// 3. collections/index.mcdl（若存在）
```

**内置库特例：**

对于标准库中的核心模块，编译器**可选**支持无扩展名包含：

```mcdl
// 特殊情况：内置标准库模块
include "math"      // 等价于 include "math.mcdl"（仅限内置标准库）
include "logger"    // 等价于 include "logger.mcdl"（仅限内置标准库）

// 用户代码必须使用扩展名
include "my_utils"  // ❌ 错误
include "my_utils.mcdl"  // ✅ 正确
```

**实现建议：**

编译器维护内置库名称白名单：

```python
BUILTIN_MODULES = {
    'math', 'logger', 'assert', 'unittest',
    'collections', 'bitlib', 'coord', 'timelib'
}


def resolve_search(include_path: str, search_paths: list[Path]) -> Path:
    # 检查是否为内置库名称（无扩展名）
    if include_path in BUILTIN_MODULES:
        stdlib_file = search_paths[0] / f"{include_path}.mcdl"
        if stdlib_file.exists():
            return stdlib_file.resolve()

    # 常规路径解析
    # ...
```

### 3. 项目结构约定

#### 3.1 推荐目录结构

本提案**推荐**（但不强制）以下项目结构：

```
my_project/
├── src/                    # 用户源代码
│   ├── main.mcdl           # 主入口文件
│   ├── game/
│   │   ├── player.mcdl
│   │   └── world.mcdl
│   └── utils/
│       └── helpers.mcdl
├── lib/                    # 第三方库（可选）
│   └── external_lib/
│       └── ...
├── tests/                  # 测试文件（可选）
│   └── test_game.mcdl
├── docs/                   # 项目文档（可选）
│   └── README.md
├── hook/                   # 编译钩子（可选）
│   ├── pre_build.sh
│   └── post_build.sh
└── build/                  # 编译输出（工具生成）
    └── datapack/
```

#### 3.2 目录职责说明

| 目录       | 职责      | 约定                   |
|----------|---------|----------------------|
| `src/`   | 项目源代码   | 默认搜索路径之一             |
| `lib/`   | 第三方库/依赖 | 可通过 `-I lib` 添加到搜索路径 |
| `tests/` | 测试代码    | 不参与生产编译              |
| `build/` | 编译产物    | 由构建工具管理，用户可忽略        |
| `docs/`  | 项目文档    | 不参与编译                |
| `hook/`  | 编译钩子    | 由构建工具在特定时机执行         |

#### 3.3 特殊文件约定

**主入口文件：**

- 默认为 `src/main.mcdl`
- 可通过命令行参数或构建工具配置指定

**模块索引文件（可选）：**

- 目录下的 `__init__.mcdl` 或 `index.mcdl` 可作为默认导出
- 支持 `include "collections"` 而非 `include "collections/stack.mcdl"`

**编译钩子（可选）：**

- `hook/pre_build.sh` 或 `hook/pre_build.mcdl`：编译前执行
- `hook/post_build.sh` 或 `hook/post_build.mcdl`：编译后执行
- 由构建工具负责调用，编译器不处理

#### 3.4 钩子系统（构建工具职责）

虽然钩子不由编译器处理，但本提案推荐以下约定：

**钩子触发时机：**

```
1. pre_build    - 编译开始前
2. pre_compile  - 每个文件编译前
3. post_compile - 每个文件编译后
4. pre_optimize - 优化开始前
5. post_optimize - 优化结束后
6. pre_package  - 打包开始前
7. post_build   - 完整构建结束后
```

**钩子文件格式：**

```bash
# hook/pre_build.sh（Shell 脚本）
#!/bin/bash
echo "Starting build..."
# 生成版本号、清理缓存等

# 或使用 MCDL 脚本（需构建工具支持）
# hook/pre_build.mcdl
fn pre_build() {
    log_info("BUILD", "Starting build process")
    // 执行预处理逻辑
}
```

## 附录 A：编译器实现参考

### include_manager.py 接口规范

```python
from pathlib import Path
from typing import Set, List

# 内置模块白名单（可选特性）
BUILTIN_MODULES = {
    'math', 'logger', 'assert', 'unittest',
    'collections', 'bitlib', 'coord', 'timelib',
    'stdbool', 'errorlib', 'debuglib'
}


class IncludeManager:
    """管理文件包含的核心组件"""

    def __init__(
            self,
            project_root: Path,
            stdlib_path: Path,
            search_paths: List[Path]
    ):
        self.project_root = project_root
        self.stdlib_path = stdlib_path
        self.search_paths = [stdlib_path] + search_paths

        self._included_files: Set[Path] = set()
        self._include_stack: List[Path] = []

    def resolve(self, current_file: Path, include_path: str) -> Path:
        """
        解析 include 路径为绝对路径
        
        Args:
            current_file: 当前正在处理的文件路径
            include_path: include 语句中的路径字符串
        
        Returns:
            解析后的规范绝对路径
        
        Raises:
            FileNotFoundError: 文件未找到
        """
        # 相对路径
        if include_path.startswith('./') or include_path.startswith('../'):
            return self._resolve_relative(current_file, include_path)

        # 绝对路径
        if include_path.startswith('/'):
            return self._resolve_absolute(include_path)

        # 搜索路径
        return self._resolve_search(include_path)

    def should_include(self, file_path: Path) -> bool:
        """
        判断文件是否应该包含（去重检查）
        
        Args:
            file_path: 文件的规范绝对路径
        
        Returns:
            True 若文件未被包含过，False 否则
        """
        canonical = file_path.resolve()
        return canonical not in self._included_files

    def mark_included(self, file_path: Path):
        """标记文件已包含"""
        self._included_files.add(file_path.resolve())

    def enter_file(self, file_path: Path):
        """
        进入文件处理（用于循环依赖检测）
        
        Args:
            file_path: 文件路径
        
        Raises:
            CircularDependencyError: 检测到循环依赖
        """
        canonical = file_path.resolve()

        if canonical in self._include_stack:
            chain = self._include_stack + [canonical]
            raise CircularDependencyError(chain)

        self._include_stack.append(canonical)

    def exit_file(self):
        """退出文件处理"""
        self._include_stack.pop()

    # 私有方法
    def _resolve_relative(self, current_file: Path, path: str) -> Path:
        base_dir = current_file.parent
        target = base_dir / path

        if target.suffix == '.mcdl' and target.exists():
            return target.resolve()

        # 可选：支持目录包含
        if target.is_dir():
            for index_name in ['__init__.mcdl', 'index.mcdl']:
                index_file = target / index_name
                if index_file.exists():
                    return index_file.resolve()

        raise FileNotFoundError(f"Cannot find {path}")

    def _resolve_absolute(self, path: str) -> Path:
        target = self.project_root / path.lstrip('/')

        if target.suffix == '.mcdl' and target.exists():
            return target.resolve()

        if target.is_dir():
            for index_name in ['__init__.mcdl', 'index.mcdl']:
                index_file = target / index_name
                if index_file.exists():
                    return index_file.resolve()

        raise FileNotFoundError(f"Cannot find {path}")

    def _resolve_search(self, path: str) -> Path:
        # 特殊处理：内置模块白名单（可选特性）
        if path in BUILTIN_MODULES:
            stdlib_file = self.stdlib_path / f"{path}.mcdl"
            if stdlib_file.exists():
                return stdlib_file.resolve()

        # 常规搜索
        for search_dir in self.search_paths:
            # 直接文件查找
            candidate = search_dir / path
            if candidate.suffix == '.mcdl' and candidate.exists():
                return candidate.resolve()

            # 目录包含（可选）
            if candidate.is_dir():
                for index_name in ['__init__.mcdl', 'index.mcdl']:
                    index_file = candidate / index_name
                    if index_file.exists():
                        return index_file.resolve()

        raise FileNotFoundError(
            f"Cannot find {path} in search paths"
        )


class CircularDependencyError(Exception):
    """循环依赖错误"""

    def __init__(self, chain: List[Path]):
        self.chain = chain
        super().__init__(self._format_chain())

    def _format_chain(self) -> str:
        names = [p.name for p in self.chain]
        return " -> ".join(names)
```
