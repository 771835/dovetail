## 1. 条件编译注解（Condition Annotations）

条件编译注解的处理时机为 **PRE_SYMBOL**。
其唯一的语义效果是决定当前声明是否被跳过，不向符号写入任何信息。
跳过时，当前声明的参数解析、符号注册、IR 生成均不执行。

---

### `@version`

**参数**

| 参数名   | 类型     | 默认值        | 说明             |
|-------|--------|------------|----------------|
| `min` | string | `"1.20.4"` | 允许编译的最低目标版本（含） |
| `max` | string | `"1.21.4"` | 允许编译的最高目标版本（含） |

**语义**  
当编译目标版本不在 `[min, max]` 闭区间内时，跳过当前声明。
版本号须为编译器已知的合法正式版本字符串，否则编译器报 `UnsupportedTargetVersion` 错误
并跳过当前声明。

---

### `@target`

**参数**

| 参数名       | 类型     | 默认值      | 合法值                                           |
|-----------|--------|----------|-----------------------------------------------|
| `edition` | string | `"java"` | `"je"`（Java Edition）/ `"be"`（Bedrock Edition） |

**语义**  
当编译目标平台与指定值不符时，跳过当前声明。

---

### `@if_not_exists`

**参数**：无

**语义**  
在当前作用域（及其父作用域）中查找与当前声明同名的符号。
若已存在同名符号，则跳过当前声明。  
典型用途是库函数提供可被用户覆盖的默认实现。

---

### `@if_symbol`

**参数**

| 参数名    | 类型     | 默认值     | 合法值                                               |
|--------|--------|---------|---------------------------------------------------|
| `name` | string | `""`    | 任意符号名                                             |
| `type` | string | `"any"` | `"any"` / `"function"` / `"class"` / `"variable"` |

**语义**  
在当前作用域中查找名为 `name` 的符号。若符号不存在，则跳过当前声明。
若符号存在但 `type` 不为 `"any"` 且符号类型与指定类型不匹配，
同样跳过当前声明。

---

### `@if_feature`

**参数**

| 参数名       | 类型     | 默认值  | 说明    |
|-----------|--------|------|-------|
| `feature` | string | `""` | 功能标识符 |

**语义**  
当编译器未启用指定 feature 时，跳过当前声明。
feature flag 系统的具体定义另行规范，当前版本此注解恒不产生跳过效果。

---

## 2. 生命周期注解（Lifecycle Annotations）

生命周期注解的处理时机为 **POST_SYMBOL**，仅允许作用于函数。
其语义是向符号写入 flags 和 metadata，由后端负责将对应函数注册到
Minecraft 数据包的执行入口。

---

### `@init`

**参数**：无  
**适用目标**：函数

**语义**  
标记此函数应在数据包加载时执行。后端须将此函数注册到数据包的 `#load` tag。
此注解产生 flags：`load_hook`、`no_dce`。
此注解产生 metadata 键：`hook_type`，值为字符串 `"load"`。

---

### `@tick`

**参数**

| 参数名        | 类型  | 默认值 | 约束     |
|------------|-----|-----|--------|
| `interval` | int | `1` | 必须为正整数 |

**适用目标**：函数

**语义**  
标记此函数应每隔 `interval` 游戏刻执行一次。后端须将此函数注册到数据包的
`#tick` tag，并按 `interval` 控制执行频率。
`interval` 不为正整数时，编译器报 `AnnotationArgumentError` 错误。
此注解产生 flags：`tick_hook`、`no_dce`。
此注解产生 metadata 键：`hook_type`（值 `"tick"`）、`interval`（值为实际传入的整数）。

---

## 3. 可见性注解（Visibility Annotations）

可见性注解的处理时机为 **POST_SYMBOL**，仅允许作用于函数。
其语义是向符号写入 flags，由优化器据此调整对目标符号的处理策略。

---

### `@internal`

**参数**：无  
**适用目标**：函数

**语义**  
标记此函数为内部实现，允许优化器对其执行激进优化。
此注解产生 flags：`aggressive_opt`。  
未标注 `@internal` 的函数，优化器应采用保守策略。

---

### `@noinline`

**参数**：无  
**适用目标**：函数

**语义**  
禁止优化器对此函数执行内联。
此注解产生 flags：`no_inline`。

---

## 4. 链接注解（Linkage Annotations）

链接注解的处理时机为 **POST_SYMBOL**，仅允许作用于函数。
链接注解均为**实验性注解**，使用时须传入 `--experimental` 编译参数。

---

### `@export`

**参数**

| 参数名    | 类型     | 默认值          | 说明                                    |
|--------|--------|--------------|---------------------------------------|
| `path` | string | `""`         | 导出路径，格式为 `namespace:path/to/function` |
| `abi`  | string | `"dovetail"` | 调用约定，合法值见下方 ABI 说明                    |

**适用目标**：函数

**语义**  
将此函数导出，使其可被外部数据包调用。
导出的函数名不经过归一化处理，因此函数名不可含大写字母或非 ASCII 字符。
此注解产生 flags：`no_dce`、`preserve_name`。
此注解产生 metadata 键：`abi`、`path`。

当 `abi` 为 `"clang-mc"` 时，编译器须对函数的全部参数类型及返回值类型进行
FFI 安全性校验，不合法的类型须报 `NotFFISafeType` 错误。详见 ABI 说明。

---

### `@extern`

**参数**

| 参数名    | 类型     | 默认值          | 说明                                    |
|--------|--------|--------------|---------------------------------------|
| `path` | string | `""`         | 导入路径，格式为 `namespace:path/to/function` |
| `abi`  | string | `"dovetail"` | 调用约定，合法值见下方 ABI 说明                    |

**适用目标**：函数

**语义**  
声明一个由外部数据包提供实现的函数。带有此注解的函数不得提供函数体，
声明形式须为 `fn name(...);`。
此注解将函数类型覆盖为 `EXTERN`，产生 flags：`no_inline`、`no_dce`，
产生 metadata 键：`abi`、`path`。
同样在 `abi` 为 `"clang-mc"` 时执行 FFI 安全性校验。

---

### ABI 说明

| 值            | 说明                                  |
|--------------|-------------------------------------|
| `"dovetail"` | Dovetail 原生调用约定，支持完整类型系统            |
| `"clang-mc"` | 与 clang-mc 编译产物兼容的调用约定，仅支持 FFI 安全类型 |

**FFI 安全类型（`abi="clang-mc"` 时有效）**

以下类型可在 `clang-mc` ABI 下跨边界传递，其余类型均不允许：

| 类型         | 允许 |
|------------|----|
| `int`      | ✅  |
| `bool`     | ✅  |
| `void`     | ✅  |
| `string`   | ❌  |
| 类实例及其他复合类型 | ❌  |

## 5. 后端提示注解（Backend Hint Annotations）

后端提示注解的处理时机为 **POST_SYMBOL**。
此类注解不影响 IR 生成，仅通过 flags 向后端传递实现提示。

---

### `@recursive`

**参数**：无  
**适用目标**：函数

**语义**  
提示后端此函数存在递归调用。后端须为此函数的局部变量使用栈式存储策略，
而非默认的全局计分板策略，以保证递归调用时数据不被覆盖。
此注解产生 flags：`allow_recursion`、`no_inline`。

---

## 6. 元数据注解（Metadata Annotations）

元数据注解不影响代码生成，其处理结果仅作为 metadata 附加在符号上，
供文档生成工具、IDE 插件等工具链消费。

`@deprecated` 的处理时机为 **PRE_SYMBOL**，其余元数据注解的处理时机为
**POST_SYMBOL**。

---

### `@deprecated`

**参数**

| 参数名   | 类型     | 默认值  |
|-------|--------|------|
| `msg` | string | `""` |

**语义**  
标记此声明为已废弃。此注解产生 metadata 键：`deprecated_msg`。  
当编译器开启 `--disable-deprecated-function` 选项时，此注解产生跳过效果，
即带有此注解的函数不参与编译。

---

### `@doc`

**参数**

| 参数名    | 类型     | 说明                     |
|--------|--------|------------------------|
| `text` | string | 文档字符串，建议使用 Markdown 格式 |

**语义**  
为目标符号附加文档说明。此注解产生 metadata 键：`doc`。

---

### `@author`

**参数**

| 参数名    | 类型     | 说明   |
|--------|--------|------|
| `name` | string | 作者名称 |

**语义**  
标记目标符号的作者。此注解产生 metadata 键：`author`。

---

### `@since`

**参数**

| 参数名       | 类型     | 说明        |
|-----------|--------|-----------|
| `version` | string | 引入此符号的版本号 |

**语义**  
标记目标符号首次引入的版本。此注解产生 metadata 键：`since`。
