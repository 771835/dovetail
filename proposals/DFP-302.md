## 摘要

本提案定义 Dovetail 标准库的错误处理模式和最佳实践，在无异常机制的语言约束下，提供统一的错误检测和报告方案。

---

## 范围界定

本提案仅处理 **运行时错误**（Runtime Error），包括：

- 函数输入验证失败
- 数值运算溢出
- 数组越界访问
- 资源耗尽

编译期错误由 [DFP 7: 编译期错误恢复及报告机制](/proposals/DFP-7.md) 覆盖。

---

## 动机

当前问题：

1. 函数遇到非法输入返回魔数（-1, 0），语义不明确
2. 无法区分"正常返回 0"和"错误返回 0"
3. 错误信息无法传播到调用者
4. 调试困难，无运行时检查

示例：

```mcdl
// 当前实现：错误难以检测
let result = isqrt(-5)  // 返回 0，但 0 也是合法值
if (result == 0) {
    // 无法判断是输入为 0 还是输入非法
}
```

## 技术规范

### 1. 错误处理模式

由于 Dovetail 暂无异常机制，采用以下模式：

#### 模式 1：返回值 + 全局错误码

```mcdl
// errorlib.mcdl - 错误处理核心库

let ERRNO_OK = 0                    // 无错误
let ERRNO_INVALID_ARG = 1           // 非法参数
let ERRNO_OUT_OF_RANGE = 2          // 超出范围
let ERRNO_OVERFLOW = 3              // 溢出
let ERRNO_UNDERFLOW = 4             // 下溢
let ERRNO_DIV_BY_ZERO = 5           // 除零错误
let ERRNO_NOT_FOUND = 6             // 未找到
let ERRNO_ALREADY_EXISTS = 7        // 已存在

// 全局错误码（类似 C 的 errno）
let g_errno = 0

/**
 * 设置全局错误码
 */
fn set_errno(error_code: int) {
    g_errno = error_code
}

/**
 * 获取并清除错误码
 */
fn get_and_clear_errno() -> int {
    let err = g_errno
    g_errno = ERRNO_OK
    return err
}

/**
 * 检查是否有错误
 */
fn has_errno() -> boolean {
    return g_errno != ERRNO_OK
}
```

#### 模式 2：Option 类型模拟（使用哨兵值）

```mcdl
// 对于可能失败的函数，定义明确的失败值
let INVALID_INDEX = -1
let INVALID_COORD = 2147483647      // INT_MAX 作为哨兵

fn find_element(arr: array<int>, target: int) -> int {
    // 返回索引，未找到时返回 INVALID_INDEX
    for (let i = 0; i < array_length(arr); i += 1) {
        if (arr[i] == target) {
            return i
        }
    }
    return INVALID_INDEX
}

// 使用示例
let index = find_element(my_array, 42)
if (index != INVALID_INDEX) {
    // 找到了
} else {
    // 未找到
}
```

#### 改进后（带错误处理）

```mcdl
include "errorlib"

/**
 * 快速幂（带错误检查）
 * 
 * 参数:
 *   base - 底数
 *   exp - 指数（必须非负）
 * 
 * 返回值:
 *   计算结果，失败时返回 0 并设置 g_errno
 * 
 * 错误码:
 *   ERRNO_INVALID_ARG - exp < 0
 *   ERRNO_OVERFLOW - 结果溢出（可选检测）
 */
 fn pow(base: int, exp: int) -> int {
    // 输入验证
    if (exp < 0) {
        set_errno(ERRNO_INVALID_ARG)
        return 0
    }
    
    // 溢出检测（可选，根据性能需求）
    if (abs(base) > 1 && exp > 30) {
        set_errno(ERR_OVERFLOW)
        return 0
    }
    
    // 正常计算
    let result = 1
    while (exp > 0) {
        if (exp % 2 == 1) {
            result = result * base
        }
        base = base * base
        exp = exp / 2
    }
    
    return result
}
 
// 使用示例
fn main() {
    let result = pow(2, 10)
    if (has_error()) {
        let err = get_errno()
        if (err == ERR_INVALID_ARG) {
            print("错误：指数不能为负数")
        } else if (err == ERR_OVERFLOW) {
            print("错误：结果溢出")
        }
        clear_error()
    } else {
        print(f"结果：{result}")
    }
}
```

### 4. 使用示例

#### 示例 1：数组访问安全封装

```mcdl
include "errorlib"
include "debuglib"

fn array_get_safe(arr: array<int>, index: int) -> int {
    if (!check_bounds(index, 0, array_length(arr) - 1, "index")) {
        return 0  // 返回默认值
    }
    return arr[index]
}

// 使用
let value = array_get_safe(my_array, 10)
if (has_error()) {
    print("数组访问越界！")
}
```

#### 示例 2：链式错误传播

```mcdl
fn complex_calculation(x: int) -> int {
    let step1 = pow(x, 2)
    if (has_error()) return 0
    
    let step2 = isqrt(step1)
    if (has_error()) return 0
    
    return step2
}
```

## 最佳实践

### 1. 函数设计原则

- 所有可能失败的函数应设置错误码
- 文档明确列出所有可能的错误码
- 使用明确的哨兵值（避免魔数）
- 提供"安全版本"和"快速版本"两种 API
- 其他错误记录方法应在文档中完整标注

### 2. 调用者责任

- 调用可能失败的函数后检查错误
- 及时清除错误码（避免误判）
- 在调试模式启用断言

### 3. 错误恢复策略

- 返回默认值（如 0、空字符串）
- 跳过当前操作继续执行
- 记录日志并通知管理员
