## 摘要

本提案定义 Dovetail 标准库中的常量命名规范和组织结构，提供 Minecraft 相关的常量定义库，减少代码中的魔数，提高可读性和可维护性。

---

## 动机

当前问题：

1. 代码中充斥魔数（如 `24000` 表示一天，`20` 表示 TPS）
2. 跨版本兼容困难（如世界高度从 256 改为 384）
3. 硬编码值分散，修改时容易遗漏

改进后：

```mcdl
// 改进前
if (y >= -64 && y <= 319) { ... }

// 改进后
if (y >= WORLD_MIN_Y && y <= WORLD_MAX_Y) { ... }
```

---

## 技术规范

### 1. 常量定义规范

#### 1.1 命名约定

- 使用 `UPPER_SNAKE_CASE`（全大写下划线分隔）
- 带单位的常量应包含单位后缀（如 `_TICKS`, `_BLOCKS`）
- 版本相关常量应包含版本号（如 `_V1_18`）

#### 1.2 组织结构

```mcdl
// constants/
//   ├── game_constants.mcdl    - 游戏机制常量
//   ├── world_constants.mcdl   - 世界/坐标相关
//   ├── time_constants.mcdl    - 时间相关
//   ├── entity_constants.mcdl  - 实体相关
//   └── item_constants.mcdl    - 物品相关
```

## 使用示例

### 示例 1：时间转换函数使用常量

```mcdl
include "constants/game_constants"

fn is_daytime(time: int) -> boolean {
    let daytime = mod(time, TICKS_PER_DAY)
    return daytime >= DAYTIME_START && daytime < DAYTIME_END
}

fn ticks_to_seconds(ticks: int) -> int {
    return ticks / TICKS_PER_SECOND
}
```

### 示例 2：坐标验证

```mcdl
include "constants/world_constants"

fn is_valid_overworld_y(y: int) -> boolean {
    return y >= WORLD_MIN_Y && y <= WORLD_MAX_Y
}

fn clamp_to_build_height(y: int) -> int {
    return clamp(y, WORLD_MIN_Y, WORLD_MAX_Y)
}
```

### 示例 3：实体属性判断

```mcdl
include "constants/entity_constants"

fn is_player_low_health(health: int) -> boolean {
    return health < PLAYER_MAX_HEALTH / 2
}
```

## 版本兼容策略

### 策略 1：编译时选择

通过编译器配置选择目标版本：

```mcdl
#if MINECRAFT_VERSION >= "1.18"
    let WORLD_MIN_Y = -64
#else
    let WORLD_MIN_Y = 0
#endif
```

### 策略 2：多版本常量库

```
constants/
  ├── v1_17/
  ├── v1_18/
  └── v1_21/
```
