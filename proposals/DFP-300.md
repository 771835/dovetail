## 摘要

本提案规定了Dovetail语言的完整内置函数体系，分为**通用内置函数**和**后端提供函数**
两大类。通用内置函数通过IR指令直接映射到虚拟机操作，提供跨平台的核心能力；后端提供函数由各平台后端实现，提供Minecraft
Java版、基岩版等平台特定功能。

## 动机

1. **统一用户体验**：使用者无需区分函数来源，所有内置函数在一个文档中查阅
2. **开发效率**：提供常用功能无需开发者重复实现
3. **语言完整性**：完善基础类型操作与平台能力
4. **性能优化**：通过IR指令直译避免中间层开销
5. **规范统一**：建立标准化的函数命名和行为准则

## 技术规范

### 第一部分：通用内置函数

通用内置函数由编译器前端/IR层实现，与具体后端无关，保证跨平台一致性。

#### 1.1 类型转换函数

| 函数签名                      | 功能描述      | 特殊行为         |
|---------------------------|-----------|--------------|
| `int(value: string): int` | 将字符串转换为整数 | 输入数据异常时，无法检测 |
| `str(value: int): string` | 将整数转换为字符串 | -            |

**说明**：

- `int()`在遇到非法字符串时行为未定义，建议配合数据验证使用
- 未来版本可能扩展支持浮点数转换

#### 1.2 实用函数

| 函数签名                          | 功能描述     | 返回值说明     |
|-------------------------------|----------|-----------|
| `type_of(value: any): string` | 返回值的类型名称 | 如`"int"`等 |
| `swap(a: any, b: any): void`  | 交换两个变量的值 | 无返回值      |

**说明**：

- `swap()`通过引用语义实现，修改变量自身而非副本
- `type_of()`返回的类型名称遵循 DFP-1 定义的类型系统

#### 1.3 字符串处理函数

| 函数签名                                        | 功能描述      | 输出目标    |
|---------------------------------------------|-----------|---------|
| `print(msg: string): void`                  | 输出字符串到控制台 | 编译器标准输出 |
| `strcat(dest: string, src: string): string` | 拼接两个字符串   | 返回新字符串  |

**说明**：

- `print()`的输出目标由后端决定（如tellraw、日志文件等）
- `strcat()`不修改原字符串，返回新拼接结果

#### 1.4 数学函数

| 函数签名                       | 功能描述  | 特殊行为       |
|----------------------------|-------|------------|
| `abs(x: int): int`         | 返回绝对值 | 最小负数返回本身   |
| `min(a: int, b: int): int` | 返回较小值 | 相等时返回第一个参数 |
| `max(a: int, b: int): int` | 返回较大值 | 相等时返回第一个参数 |

**说明**：

- 未来版本计划支持浮点数版本
- 最小负数（INT_MIN）的`abs()`返回自身以避免溢出

#### 1.5 语言层指令

| 函数签名                          | 功能描述            | 特殊行为           | 
|-------------------------------|-----------------|----------------|
| `_call(scope: string): void`  | 插入跳转指令，跳转到指定作用域 | 作用域不存在时编译错误    |
| `exec(command: string): void` | 执行平台原生命令        | 命令语法错误时行为由后端决定 |

**警告**：

- `_call()`直接操作控制流，**不应作为递归的替代品**，存在栈溢出风险
- `exec()`绕过类型检查，使用时需确保命令字符串合法性

### 第二部分：后端提供函数

后端提供函数由各平台后端实现，提供游戏引擎特定能力。不同后端可能有不同的实现和限制。

#### 2.1 Java版函数

##### 2.1.1 游戏交互函数

| 函数签名                                                                                 | 功能描述          | 特殊行为                                  |
|--------------------------------------------------------------------------------------|---------------|---------------------------------------|
| `tellraw_text(target: string, msg: string): void`                                    | 向目标发送文本消息     | 消息中包含未转义的`'`或`"`时执行失败                 |
| `tellraw_json(target: string, json: string): void`                                   | 向目标发送JSON格式消息 | JSON格式错误时执行失败                         |
| `setblock(x: int, y: int, z: int, block_id: string, mode: string = "replace"): void` | 设置指定坐标的方块     | mode可选值: `replace`, `destroy`, `keep` |
| `get_block(x: int, y: int, z: int): string`                                          | 获取指定坐标的方块类型   | 返回方块ID字符串                             |

##### 2.1.2 物品与实体管理

| 函数签名                                                                        | 功能描述   | 默认值说明       |
|-----------------------------------------------------------------------------|--------|-------------|
| `item_spawn(x: int, y: int, z: int, item_id: string, count: int = 1): void` | 生成掉落物品 | count默认为1   |
| `give(target: string, item_id: string, count: int = 1): void`               | 给予玩家物品 | count默认为1   |
| `summon(entity_id: string, x: int, y: int, z: int, nbt: string = ""): void` | 生成实体   | nbt为可选NBT数据 |
| `kill(target: string): void`                                                | 清除目标实体 | -           |

##### 2.1.3 世界控制函数

| 函数签名                                                                                 | 功能描述   | 参数说明                                     |
|--------------------------------------------------------------------------------------|--------|------------------------------------------|
| `tp(target: string, x: int, y: int, z: int): void`                                   | 传送目标实体 | -                                        |
| `time_set(value: int): void`                                                         | 设置世界时间 | 0-24000范围                                |
| `weather(type: string): void`                                                        | 设置天气   | 可选: `clear`, `rain`, `thunder`           |
| `difficulty(level: string): void`                                                    | 设置难度   | 可选: `peaceful`, `easy`, `normal`, `hard` |
| `gamerule(rule: string, value: string): void`                                        | 更改游戏规则 | rule为游戏规则名，value为对应值                     |
| `fill(x1: int, y1: int, z1: int, x2: int, y2: int, z2: int, block_id: string): void` | 区域填充方块 | 填充从(x1,y1,z1)到(x2,y2,z2)的区域              |

##### 2.1.4 玩家效果与属性

| 函数签名                                                                                   | 功能描述    | 默认值说明           |
|----------------------------------------------------------------------------------------|---------|-----------------|
| `effect(target: string, effect: string, duration: int = 30, amplifier: int = 0): void` | 添加状态效果  | 持续时间默认30秒，等级默认0 |
| `attribute(target: string, attr: string, value: int): void`                            | 修改实体属性  | -               |
| `damage(target: string, amount: int, source: string = "generic"): void`                | 对目标造成伤害 | 伤害来源默认为generic  |

##### 2.1.5 高级功能

| 函数签名                                                                              | 功能描述     | 参数说明                          |
|-----------------------------------------------------------------------------------|----------|-------------------------------|
| `tag(target: string, tag: string): void`                                          | 管理实体标签   | 添加或移除标签（具体操作由后端实现决定）          |
| `scoreboard(op: string, target: string, objective: string, value: int = 0): void` | 记分板操作    | op可选: `set`, `add`, `remove`等 |
| `bossbar(op: string, id: string, value: string = ""): void`                       | Boss血条控制 | op可选: `add`, `remove`, `set`等 |
| `randint(min: int, max: int): int`                                                | 生成随机整数   | 返回[min, max]范围内的随机数           |

##### 2.1.6 执行命令函数

| 函数签名                                                            | 功能描述          | 特殊行为      |
|-----------------------------------------------------------------|---------------|-----------|
| `exec(command: string): void`                                   | 执行给定单行命令      | 命令错误不进行执行 |
| `execute(function: string, subcmd: string, args: string): void` | 执行连锁execute命令 | 支持复杂条件执行链 |

**说明**：

- `execute()`支持Minecraft execute命令的完整语法，允许构建复杂的条件执行链
- 推荐使用封装好的高层函数而非直接使用`exec()`

## 附录A：函数设计原则

1. **最小惊讶原则**：函数行为应符合开发者直觉
2. **平台一致性**：通用函数在所有后端保持相同语义
3. **安全优先**：避免未定义行为，明确错误处理
4. **性能考量**：优先使用IR指令实现，减少运行时开销
